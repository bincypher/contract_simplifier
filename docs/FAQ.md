## General

### What is Clarity?

Clarity is an AI-powered document analysis platform that helps users understand legal and policy documents in plain English. It analyzes contracts, agreements, and policies to identify risks, obligations, key points, and missing information — without providing legal advice.

### Is Clarity a legal advice tool?

**No.** Clarity provides educational document explanations only. It is not a substitute for professional legal advice. The application explicitly states this in multiple places:
- The main disclaimer below the upload form
- The chat interface disclaimer
- The footer
- The AI system prompts (the AI is instructed to never provide legal advice)

Always consult a qualified professional for important legal decisions.

### What types of documents can I analyze?

Clarity supports the following document types:

| Supported Types |
|---|
| Insurance Policy |
| Employment Contract |
| Rental Agreement |
| NDA (Non-Disclosure Agreement) |
| Service Agreement |
| Vendor Contract |
| Purchase Order |
| Privacy Policy |
| Terms & Conditions |
| Loan Agreement |
| Legal Notice |
| Other Legal or Policy Document |

Documents that are not legal or policy-related (e.g., textbooks, resumes, marketing materials) will be rejected during the eligibility check.

### What file formats are supported?

| Format | Extension | Max Size |
|---|---|---|
| PDF | `.pdf` | 15 MB, 1–100 pages |
| JPEG | `.jpg`, `.jpeg` | 15 MB, max 25 MP |
| PNG | `.png` | 15 MB, max 25 MP |
| WebP | `.webp` | 15 MB, max 25 MP |

> **⚠️ Note:** DOCX and TXT files are **not supported** in the current version.

### Does Clarity support multiple languages?

Currently, Clarity is designed for **English-language documents**. The UI is English-only, and the AI analysis is optimized for English text. Multi-language support is on the future roadmap.

## Document Upload

### Can I upload scanned PDFs?

**Yes.** Clarity handles scanned PDFs in two ways:
1. **PDFs with extractable text** — Text is extracted using `pdf-parse` and sent to the AI as plain text.
2. **Image-based PDFs** (scanned, no extractable text) — The PDF is sent as a base64-encoded file to OpenAI's Vision API for analysis.

### Why is my PDF rejected with "corrupted or encrypted"?

Common reasons:
- The PDF is **password-protected** — remove the password and re-upload.
- The PDF is **corrupted** — try re-saving it from a PDF editor.
- The PDF uses **non-standard encoding** — try exporting it from a different application.

### Can I upload multiple documents at once?

**No.** Clarity processes one document at a time. Each upload replaces the previous one.

### What happens to my uploaded file?

Your file is:
1. Read into memory on the server.
2. Validated (type, size, integrity).
3. Processed by the AI.
4. **Discarded** — it is not stored on any server.

The file exists in memory **only** during the request processing time.

## Analysis

### How does the risk score work?

The risk score is a **0–100 scale** where:
- **0–33:** Low risk (document has favorable or standard terms)
- **34–66:** Medium risk (some concerning clauses or missing protections)
- **67–100:** High risk (significant concerns requiring attention)

The score is generated by the AI based on evidence from the document. It considers:
- Financial exposure and obligations
- Hidden conditions and exclusions
- Termination and renewal terms
- Privacy and data handling
- Penalties and missing information

### How accurate is the analysis?

Each analysis includes a **confidence percentage** reflecting the AI's certainty in its classification and findings. The analysis uses:
- **Temperature 0** for classification (deterministic).
- **Temperature 0.1** for analysis (near-deterministic).
- **Structured JSON Schema** output to prevent hallucination.
- **Zod validation** to ensure response integrity.

However, AI analysis is not perfect. Always verify critical findings with the original document and consult a professional.

### Why was my document classified as "Unsupported"?

The eligibility classifier must determine with ≥ 70% confidence that the document is a legal or policy document. Rejection reasons include:
- The document is educational, marketing, or technical content.
- The document is a resume, invoice, or receipt without contractual terms.
- The document is ambiguous and the classifier could not determine its type.

### Can I export the analysis report?

**Not in the current version.** Report export (PDF download) is a planned future feature. As a workaround, you can use your browser's print function (Ctrl+P / Cmd+P) to save the page as a PDF.

## Document Chat

### How does the Document Q&A work?

After an analysis is complete, you can ask questions about the uploaded document. The system:

1. **Checks your question** for prompt injection attempts (regex patterns).
2. **Classifies relevance** — an AI call determines if the question is about the document.
3. **Generates an answer** — a second AI call answers the question using only the document content.
4. **Provides evidence** — direct excerpts from the document are included with page numbers.

### Why was my question rejected?

Questions are rejected if they:
- Are not related to the uploaded document (e.g., general knowledge, math, coding).
- Attempt to reveal system prompts, API keys, or hidden instructions.
- Attempt to bypass or override the system's guardrails.
- Ask for legal conclusions beyond explaining what the document says.
- Are too ambiguous for the system to classify.

### How long is a chat session active?

Chat sessions are valid for **30 minutes** after the document analysis. After expiration, you must analyze the document again to start a new session.

### Can I continue a conversation across sessions?

**No.** Chat history is stored in the browser's memory (React state). It is lost when:
- The page is refreshed.
- The document session expires.
- The browser tab is closed.

### How many questions can I ask?

Within the 30-minute session window, you can ask up to **30 questions** per 10-minute period (per IP address). There is no total limit beyond the session TTL.

## Security & Privacy

### Is my document stored on any server?

**No.** Documents are processed entirely in-memory and are never written to disk or stored in any database. The file exists on the server only during the duration of the HTTP request.

### Is my document sent to third parties?

**Yes — to OpenAI.** Document content is sent to the OpenAI API for AI processing. Review [OpenAI's data usage policies](https://openai.com/policies/api-data-usage-policies/) for your organization's compliance requirements.

### Is the application secure against prompt injection?

Clarity implements a **three-tier defense**:
1. **Regex detection** — Common injection patterns are blocked before any AI call.
2. **AI relevance gate** — A dedicated AI call classifies the question's intent.
3. **Grounded system prompt** — The AI is instructed to treat all input as untrusted data.

While no defense is perfect, this multi-layer approach significantly reduces the risk.

### How are document session tokens secured?

Tokens are:
- **HMAC-SHA256 signed** with a server-side secret.
- **Time-limited** (30-minute TTL).
- **Fingerprint-bound** — the document's SHA-256 hash is verified on every chat request.
- **Compared timing-safely** to prevent timing attacks.

## Technical

### What AI model is used?

By default, Clarity uses OpenAI's **gpt-4o-mini** model. This can be changed via the `OPENAI_MODEL` environment variable. The model must support:
- Chat completions API
- Structured output (`response_format: json_schema`)
- Vision capability (for image documents)

### How many API calls does each operation make?

| Operation | API Calls | Purpose |
|---|---|---|
| Document Analysis | 2 | 1 eligibility + 1 analysis |
| Document Chat Question | 2 | 1 relevance + 1 answer |

### Why is the document re-uploaded for each chat question?

This is a deliberate **security design choice**. By not storing files on the server:
- No server-side file storage is needed.
- No file cleanup or expiration logic is required.
- Document integrity is verified on every request via SHA-256 fingerprinting.
- The system remains fully stateless.

### Can I use a different AI provider?

The current implementation is tightly integrated with the OpenAI SDK and uses OpenAI-specific features (structured output with `json_schema`, vision API). Switching to a different provider would require significant modifications to the API route handlers and schema definitions.

### What happens if the OpenAI API is down?

If the OpenAI API is unreachable, the application will return:
- **HTTP 503** — if the connection cannot be established.
- **HTTP 504** — if the API times out (75-second limit).

Users should check [status.openai.com](https://status.openai.com/) for API availability.

## Deployment

### Can I deploy Clarity to Vercel?

**Yes.** Clarity is a standard Next.js application and is fully compatible with Vercel. See [DEPLOYMENT.md](DEPLOYMENT.md) for detailed instructions.

### Can I run Clarity with Docker?

The repository does not include a Dockerfile, but one can be easily created. See the recommended Dockerfile in [DEPLOYMENT.md](DEPLOYMENT.md#docker-deployment).

### Do I need a database?

**No.** Clarity is entirely stateless and does not require any database.

### How much does it cost to run?

The primary running cost is **OpenAI API usage**. Each document analysis requires 2 API calls, and each chat question requires 2 API calls. With `gpt-4o-mini`, costs are typically low per operation. Check [OpenAI's pricing page](https://openai.com/pricing) for current rates.
